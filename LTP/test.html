>>FAQ<< not sure what to do if the models stall?

In this task, you will be engaged in conversation with two different chatbot/AI models. For each message you send, you will receive two responses back: one from Model A, and one from Model B. You will then choose which of the model's responses was better overall.

  • When both choices are equally horrible, you can select one of the "Unsure" options. When both are bad, but one is still better, you can prefer that one.

  • If both responses follow the request exactly the same way and the response is exactly what you asked for, then you would select 'negligibly better.' If it followed all the rules and probably couldn't be written any better, you should select 'perfect'.

  • Please wait until both models have finished responding (the chat bubbles will turn green) to select the best response.

  • You can end the conversation by pressing "Submit" (in the lower right hand corner) whenever it feels natural to you (usually this is when you're significantly changing the topic), but at a maximum of 10 rounds. It's completely up to you how long your conversations are.

New Version: Coding Help & Questions

    • Please avoid asking for HTML, CSS, or JS. Additionally, conversations should not focus on markup languages. Also note anything inside < > will disappear; you can ask about languages that use the angle brackets, but try to avoid code that requires their use.

  • As far as what you ask of the chatbots -- they're likely to be relatively basic when asked to deal with code, so you can start easy and ratchet up the difficulty.

    • Examples:
      Write a function that does [XYZ]

      Explain this piece of code: [XYZ]

      Find bugs in the code below: [XYZ]

      Why am I not passing the test in this piece of code [XYZ]?

  • This may seem obvious to some, but we often get questions about this: you must consider which is the BEST answer, not merely select the first one that seems good enough. This is because the relative usefulness/correctness of an answer is how the model is trained - it needs to know which is the best answer. As a bit of context, it’s learning by comparing the responses. It knows that one is better than the other because you’ve told it so, so it will compare and eventually figure out why one is better than the other.

COMMON MISTAKES

  • If you're new to this project (or chat-style projects in general), DO NOT JUST START WORKING ON THIS WITHOUT READING INSTRUCTIONS.

  • You should not have a library or document of prompts that you cycle through, or anything where it becomes “formulaic.”

    • If you find yourself pasting a message into the message window, you're going too far.

    • Also, avoid using the same strategy again and again - you should vary the difficulty and details of the prompt each time you use the strategy by changing the parameters. If you would like to learn more about parameters, see Parameters.

  • Your conversations should focus on CODE. For example, don't ask it how to build a home network. The conversation must include code in at least one turn.

  • TEST THE CODE. The chatbots are very good at presenting something that looks correct but ultimately is not.

  • The focus of this task is on the coding, but you should pay attention to how the code is presented. This is especially true if the code is roughly equal in quality.

    • Are the chatbots repeating the same phrases over and over in your conversation? For example, they might start every message with "Ah, the ..."

    • Are emojis being used in every response? Does it make sense to, or does it seem like they're following a template? End users might see this as odd and as a sign of a low quality coding assistant.

    • However, this doesn't mean that if you notice a lot of emojis or similar phrasing across all conversations that you should focus on them and punish them whenever you see them in a new conversation. Just be on the lookout for repetitiveness in your conversations.

  • Anything inside < > will disappear -- so avoid conversations that are likely to lead to the chatbots using code like this (HTML, etc.) and don't use them yourself.

IMPORTANT. If you don't read and understand the the purpose of these checkboxes, you will get removed from the projects.

  • These situations will not be very common because you are NEVER purposely asking for something unsafe or harmful in this project. For example, you asked how to make a baseball bat and you get a response saying that it cannot help you because it would be unethical (or against the law, etc.).

  • In these cases, they think you're asking for something they're not allowed to help with because doing so would be unsafe or harmful. Because you're NEVER purposely asking for something unsafe or harmful in this project, these will always be mistakes where they misread your intentions. Usually these responses are along the lines of "I'm sorry, but I cannot help you with that because...". In short, they think you're asking for something related to the LIST OF TOPICS YOU SHOULD AVOID, but you of course aren't.

  • We aren't asking you to purposely target these situations, we just want to know when these happen in the course of your normal conversations. For example, don't play around with keywords that are sometimes used in unsafe contexts to see if you can fool them into making these mistakes.

  • DO NOT use these checkboxes when the models say they cannot do something for reasons unrelated to being safe or harmless. For example, if they say they don't have information about a celebrity because of their knowledge cutoff date, you DO NOT use these checkboxes.
